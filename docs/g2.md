# Assignment 6: Syntax Analysis using CFG

## Problem Statement
Write a C program that performs syntax analysis of a simple language using CFG rules.

## Theory

### Syntax Analysis (Parsing)
Syntax analysis is the second phase of compilation that checks if the tokens form a valid syntax according to the grammar rules.

### Types of Parsers
1. **Top-Down**: Start from start symbol and expand to match input
2. **Bottom-Up**: Start from input and reduce to start symbol

### Recursive Descent Parsing
A top-down parsing technique where each non-terminal has a corresponding function:
- Simple to implement
- Works for LL(1) grammars
- Uses recursion to handle nested structures

### Grammar Used
```
E → T + E | T      (Expression)
T → F * T | F      (Term)
F → ( E ) | id     (Factor)
```

This grammar handles:
- Addition (+)
- Multiplication (*)
- Parentheses ( )
- Identifiers (id)

## Parse Tree Example

For expression: `id + id * id`

```
        E
      / | \
     T  +  E
     |    /|\
     F   T * T
     |   |   |
    id   F   F
         |   |
        id  id
```

## Simulation

### Expression: `i+i*i` (where i = id)

| Step | Function | Action |
|------|----------|--------|
| 1    | E()      | Call T() |
| 2    | T()      | Call F() |
| 3    | F()      | Match id (i) |
| 4    | T()      | Return to E |
| 5    | E()      | Match + |
| 6    | E()      | Call E() recursively |
| 7    | E()→T()  | Call T() |
| 8    | T()→F()  | Call F() |
| 9    | F()      | Match id (i) |
| 10   | T()      | Match * |
| 11   | T()      | Call T() |
| 12   | T()→F()  | Match id (i) |

Result: ACCEPTED

## Code

```c
#include <stdio.h>
#include <string.h>
#include <ctype.h>

char* input;
int pos = 0;
int error = 0;

void E();
void T();
void F();

void match(char expected) {
    if(input[pos] == expected) {
        pos++;
    } else {
        printf("Error: Expected '%c', found '%c'\n", expected, input[pos]);
        error = 1;
    }
}

void F() {
    printf("F -> ");
    if(input[pos] == '(') {
        printf("(E)\n");
        match('(');
        E();
        match(')');
    } else if(isalpha(input[pos])) {
        printf("id\n");
        while(isalnum(input[pos])) pos++;
    } else {
        printf("Error in F\n");
        error = 1;
    }
}

void T() {
    printf("T -> ");
    F();
    if(input[pos] == '*') {
        printf("*T\n");
        match('*');
        T();
    }
}

void E() {
    printf("E -> ");
    T();
    if(input[pos] == '+') {
        printf("+E\n");
        match('+');
        E();
    }
}

int main() {
    char expr[100];
    
    printf("Syntax Analysis using CFG\n");
    printf("Grammar:\n");
    printf("  E -> T + E | T\n");
    printf("  T -> F * T | F\n");
    printf("  F -> ( E ) | id\n\n");
    
    printf("Enter expression (use 'i' for id, no spaces): ");
    scanf("%s", expr);
    
    input = expr;
    pos = 0;
    error = 0;
    
    printf("\nParse Tree:\n");
    E();
    
    if(error == 0 && pos == strlen(input)) {
        printf("\nResult: ACCEPTED (Valid syntax)\n");
    } else {
        printf("\nResult: REJECTED (Invalid syntax)\n");
        if(pos < strlen(input)) {
            printf("Remaining unparsed: %s\n", input + pos);
        }
    }
    
    return 0;
}
```

```

```

## Conclusion
Syntax analysis using CFG demonstrates:

1. **Grammar Enforcement**: Ensures code follows language rules
2. **Hierarchical Structure**: Builds parse tree showing expression structure
3. **Error Detection**: Identifies syntax errors early in compilation
4. **Foundation for Semantics**: Parse tree used for semantic analysis and code generation

This recursive descent parser shows how compilers validate source code structure before generating machine code.
